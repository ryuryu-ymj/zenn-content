https://github.com/bifurcationkit/BifurcationKit.jl

# 分岐解析とは
力学系の分岐の定義を軽くおさらいします．下記のような常微分方程式で表現される連続時間力学系 $F$ があるとします：

$$
\frac{\mathrm dx}{\mathrm dt} = F(x, p)
$$

$x$ は系の状態，$t$ は時間，$p$ は系の性質を変えるパラメータです．このとき，パラメータ $p$ を１つ定めると対応する力学系の解 $x(t)$ （の集合）が決まります．そして $p$ を少しずつ変化させたときに，この解 $x(t)$ の性質（例えば固定点の安定性や個数）が変化する現象を**分岐 bifurcation** と言います．

分岐解析とは，力学系 $F$ がパラメータ $p$ のどの点でどういうタイプの分岐をするのか調べる作業です．Julia を使って数値的に分岐解析を行うツールが BifurcationKit.jl です．[^AUTO]

[^AUTO]: 他の数値分岐解析ツールとしては [AUTO](https://github.com/auto-07p/auto-07p) が有名です．が，あまりにソフトウェアが古い点と利用するのに Fortran を書く必要がある点を理由に私は使うのを挫折しました．その点は BifurcationKit は Julia から利用できるモダンなパッケージで幾分使いやすいと思います．

では，数値的に分岐解析をするにはどうしたらよいでしょう？常微分方程式を長時間シミュレーションして解を求めればよいと思うかもしれませんが，様々な初期状態やパラメータに対して，それを行うのは非効率です．そこで数値分岐解析では，**解の追跡**というアプローチを取ります．解析の手順をまとめると次のようになります：

1. ある $p$ において固定点や周期軌道などの典型的な解を１つ見つける
1. $p$ を少しずつ変化させながら見つけた解を**追跡 continuation**する
1. 解の性質（安定性）が変わる分岐点を見つける
1. 分岐点から分岐する他の解の枝を見つける（1. に戻る）

以下，いくつか例を紹介しつつ，解析の手順を具体的に説明したいと思います．


# 環境
BifurcationKit は Julia というプログラミング言語のパッケージです．Julia の環境構築は以下のリンクあたりを参考にしてください：
https://julialang.org/install/
https://zenn.dev/ohno/articles/cf2b4d09d4480e

本記事では，Julia およびパッケージとして以下のバージョンを使用します：
* Julia: 1.11.5
* BifurcationKit: 0.4.16
* Plots: 1.40.13

パッケージを追加する方法は，まず，ターミナルで
```shell
julia
```
と打って，Julia の REPL（対話モード）を起動してください．そこで `]` キーを押すと，パッケージ管理モードに移行するので，
```shell
(@v1.11) pkg> add BifurcationKit, Plots
```
とすると，パッケージが追加されます．ちなみにパッケージ管理モードを抜けるには，バックスペースキーを押してください．

これ以降出てくる Julia のソースコードは，REPL に直接打ち込んで実行するか，あるいは別途ソースファイルを用意して，REPL で
```shell
julia> include("ファイル名")
```
とするか，ターミナルで
```shell
julia ファイル名
```
としてください．REPL で１行ずつ実行していく場合は，最初にパッケージをインポートすることを忘れないでください：
```shell
julia> using BifurcationKit, Plots
```


# 例１．サドルノード分岐
例として，以下の力学系を BifurcationKit で分岐解析します：

$$
\frac{\mathrm dx}{\mathrm dt} = p - x^2
$$

:::details Julia コード：サドルノード分岐の解析
```julia: SaddleNode.jl
using BifurcationKit, Plots

# 対象とする力学系
F(x, p) = p - x .^ 2
# 分岐解析として解く問題の定義
prob = BifurcationProblem(F, [1.0], [0.5], 1;
    record_from_solution=(x, p; k...) -> x[1])
# 解の追跡のパラメータ
contpar = ContinuationPar(p_min=-1.0, p_max=1.0)
# 解の追跡の実行
br = continuation(prob, PALC(), contpar; bothside=true)
# 追跡結果の出力
println("=== 追跡結果 ===")
println(br)
# 分岐図の描画
plot(br)
savefig("saddle_node.png")
# 分岐点の標準形の出力
println("=== 分岐点の標準形 ===")
println(get_normal_form(br, 2))
```
:::

コードの内容を順を追って説明します．

## Step 1. 問題の定義
まず，微分方程式の右辺を Julia の関数 `F(x, p)` として定義します．ここで，`x` や `p` はベクトルとして定義する必要があります．（あるいは名前付きタプルでもいい．）
```julia
F(x, p) = p - x .^ 2
```

続いて，`BifurcationProblem` を定義します．引数は順に：
1. 対象となる力学系，先ほどの `F`．
1. 固定点を探索する `x` の初期値．固定点が見つかるように適当に選ぶ．
1. 追跡を開始するパラメータ `p` の初期値．固定点が存在するパラメータを選ぶ．
1. 追跡を行うパラメータベクトル `p` のインデックス．`p` が名前付きタプルの場合は `(@optic _.α)` のようにする．
1. `record_from_solution`: デフォルトでは `x` のノルムを記録するようになっているので，追跡の過程で記録したい変数を定義してあげるとよい．分岐図には，この関数が返すスカラーあるいはベクトルの第１成分がプロットされる．

となっています．
```julia
prob = BifurcationProblem(F, [1.0], [0.5], 1; record_from_solution=(x, p; k...) -> x[1])
```

## Step 2. 解の追跡
解の追跡の設定を以下のように定義します．`p_min`, `p_max` で追跡を行うパラメータ `p` の範囲を決めます．
```julia
contpar = ContinuationPar(p_min=-1.0, p_max=1.0)
```

実際に追跡を行うのは以下の部分です．`PALC()` は解の追跡アルゴリズムです．`bothside` を `true` にしておくと，追跡を正負の両方向に行ってくれます．
```julia
br = continuation(prob, PALC(), contpar; bothside=true)
```

## Step 3. 結果の出力
追跡結果を出力してみましょう．
```julia
println(br)
```
```:出力結果
 ┌─ Curve type: EquilibriumCont
 ├─ Number of points: 33
 ├─ Type of vectors: Vector{Float64}
 ├─ Parameter p1 starts at 1.0, ends at 1.0
 ├─ Algo: PALC
 └─ Special points:

- #  1, endpoint at p1 ≈ +1.00000000,                                                                     step =   0
- #  2,       bp at p1 ≈ +0.00000511 ∈ (+0.00000511, +0.00109568), |δp|=1e-03, [converged], δ = (-1,  0), step =  11
- #  3, endpoint at p1 ≈ +1.00000000,                                                                     step =  32
```

`p[1] ≈ +0.00000511` で分岐点 (`bp`) が検出されました．

分岐図を描くと以下のようになります．
```julia
plot(br)
# savefig("saddle_node.png")  # 画像を保存するなら
```
![分岐図](/images/intro_bifurcation_kit/saddle_node.png)

分岐図とは分岐パラメータを横軸に，そのパラメータでの解の様子を縦軸にプロットした図のことです．上図では，太線が安定固定点，細線が不安定固定点を表し，濃い青の点が安定固定点と不安定固定点が対生成する分岐点（つまりサドルノード分岐点）を表しています．この太線と細線を合わせた曲線を１本の枝と呼び，**解の追跡とは枝を辿っていく作業**のことです．

分岐点の標準形を得るには `get_normal_form` を使います．引数には追跡結果と調べたい分岐点の番号（今回は #2 が `bp` だったので2）を指定します．
```julia
println(get_normal_form(br, 2))
```
```:出力結果
Fold bifurcation point at p1 ≈ 5.1065293785765065e-6
Normal form (aδμ + b1⋅x⋅δμ + b2⋅x²/2 + b3⋅x³/6)
┌─ a  = 1.0
├─ b1 = 0.0
├─ b2 = -2.0
└─ b3 = 0.0
```

分岐のタイプが fold 分岐（サドルノード分岐のこと）であることと，標準形が $\delta\mu - x^2$ であることがわかりました．


# 例２．ピッチフォーク分岐
先ほどのサドルノード分岐の例では分岐図の枝は１本しかありませんでしたが，枝が２本ある例を見てみましょう．以下の力学系を考えます：

$$
\frac{\mathrm dx}{\mathrm dt} = px - x^3
$$

:::details Julia コード：ピッチフォーク分岐の解析
```julia
using BifurcationKit, Plots

F(x, p) = p .* x - x .^ 3
prob = BifurcationProblem(F, [1.0], [-1.0], 1; record_from_solution=(x, p; k...) -> x[1])
contpar = ContinuationPar(p_min=-1.0, p_max=1.0)
br1 = continuation(prob, PALC(), contpar; bothside=true)
println("=== １つ目の追跡結果 ===")
println(br1)
plot(br1)
println("=== 分岐点の標準形 ===")
println(get_normal_form(br1, 2))

br2 = continuation(br1, 2; bothside=true)
println("=== ２つ目の追跡結果 ===")
println(br2)
plot!(br2)
```
:::

## Step 1. １つ目の固定点の追跡
まずは例１と同様に解の追跡を行います．
```julia
F(x, p) = p .* x - x .^ 3
prob = BifurcationProblem(F, [1.0], [-1.0], 1; record_from_solution=(x, p; k...) -> x[1])
contpar = ContinuationPar(p_min=-1.0, p_max=1.0)
br1 = continuation(prob, PALC(), contpar; bothside=true)
```

結果を出力すると，
```julia
println(br1)
println(get_normal_form(br1, 2))
plot(br1)
```

## Step 2. 分岐した別の固定点の追跡
ピッチフォーク分岐なので，分岐点から分岐する別の固定点の枝が存在するはずです．これを追跡してみましょう．

```julia
br2 = continuation(br1, 2; bothside=true)
```

`continuation` 関数に１つ目の追跡の枝と分岐点のインデックス（#2 が `bp` だったので 2）を指定すると，分岐点から分岐する枝を追跡できます．

結果を確認しましょう．
```julia
println(br2)
plot(br1, br2)  # 2つの枝を重ねてプロット
```

橙線が１つ目に追跡した枝，紫線が２つ目の枝です．このように分岐解析は**枝を追跡する → 分岐点を見つける → 分岐する他の枝を見つける → 追跡 →...** の繰り返しです．


# 例３．ホップ分岐
これまでの例はすべて固定点の分岐だったので，今度は周期軌道を追跡する例を見てみましょう．

$$
\begin{align*}
\frac{\mathrm dx}{\mathrm dt} &= lx - y + (ax - by)(x^2 + y^2) \\
\frac{\mathrm dy}{\mathrm dt} &= x + ly + (bx + ay)(x^2 + y^2)
\end{align*}
$$


## Step 1. 固定点の追跡
まずは，これまでと同様に固定点の追跡を行います．
```julia
function F(X, p)
    x, y = X
    (; l, a, b) = p  # 名前付きタプルの分割代入
    r2 = x^2 + y^2
    [
        l * x - y + r2 * (a * x - b * y),
        x + l * y + r2 * (b * x + a * y),
    ]
end

prob = BifurcationProblem(F, [1.0, 1.0],
    (a=-1.0, b=1.0, l=-1.0), (@optic _.l);  # 追跡は l について行う
    record_from_solution=(x, p; k...) -> x[1])
contpar = ContinuationPar(p_min=-1.0, p_max=1.0)
br1 = continuation(prob, PALC(), contpar; bothside=true)
```

```julia
println(br1)
println(get_normal_form(br1, 2))
```

`l ≈ +0.00044561` で超臨界ホップ分岐が起きていることがわかりました．


## Step 2. 分岐した周期軌道の追跡
続いて，ホップ分岐によって生じた周期軌道を追跡しましょう．
```julia
br2 = continuation(br1, 2, ContinuationPar(),
    PeriodicOrbitOCollProblem(20, 5))
```
ピッチフォーク分岐の例と同様に `continuation` 関数に１つ目の追跡結果と分岐点のインデックスを指定します．それからホップ分岐の場合は追加の引数として，追跡のパラメータ（`ContinuationPar()`）と周期軌道の計算手法（`PeriodicOrbitOCollProblem(20, 5)`）を指定します．周期軌道を計算手法はいくつか用意されているようですが，使い分け方は正直よくわかりません．`PeriodicOrbitOCollProblem` の引数の数を増やせば，周期軌道の近似精度が上がります．

結果を見てましょう．
```julia
println(br2)
plot(br1, br2)
```

```:出力結果
 ┌─ Curve type: PeriodicOrbitCont from Hopf bifurcation point.
 ├─ Number of points: 15
 ├─ Type of vectors: Vector{Float64}
 ├─ Parameter l starts at 0.010445609944404539, ends at 1.0
 ├─ Algo: PALC
 └─ Special points:

- #  1, endpoint at l ≈ +1.00000000,                                                                     step =  14
```

![ホップ分岐図](/images/intro_bifurcation_kit/hopf_diagram.png)
*分岐図*

紫の線が分岐点から追跡した周期軌道です．縦軸はデフォルトでは解のノルムがプロットされています．

追跡した周期軌道を１つピックアップして軌道を求めることもできます．２つ目の追跡結果 `br2` は `Number of points: 15` つまり追跡点が15個あるので，試しに10番目の周期軌道をプロットしてみましょう．
```julia
idx = 10  # 取得する周期軌道のインデックス
sol = get_periodic_orbit(br2, idx)  # 周期軌道の取得
plot(sol.t, sol.u[1, :], label="x", xlabel="time",
    title="l = $(br2[idx].param)")
plot!(sol.t, sol.u[2, :], label="y")  # plot! で重ねてプロット
```

![ホップ分岐の周期軌道の解軌道](/images/intro_bifurcation_kit/hopf_sol.png)
*解軌道*

`get_periodic_orbit` 関数に追跡結果とインデックスを指定することで，時間 `t` と軌道 `u` が入った周期軌道を取得できます．ちなみに，`sol.u[1, :]` は `sol[1, :]` と書いても同じです．
